# MCTS 动态策略引擎

这是面试 Copilot 的一个可选增强特性。开启后，系统会在实时面试辅助阶段利用蒙特卡洛树搜索（MCTS）做博弈推演，帮你提前看到"如果这样答，HR 大概率会怎么追"的路径。

### 它解决什么问题

Copilot 原有的策略树是静态的——在 Prep Phase 生成后就固定下来，只能做匹配，不能预判。MCTS 引擎在此基础上加了一层动态推演：

* 把面试看作候选人与 HR 的序贯博弈。
* 在你回答的时间窗口里（通常 30-120 秒），后台异步展开博弈树，模拟接下来 2-3 轮对话走向。
* 搜索完成后，向前端推送最优回答方向、预测追问、危险区域和当前局势胜率。

### 你会多看到什么

开启 MCTS 后，实时模式下会额外出现：

* **最优回答策略**：当前问题的推荐回答方向。
* **预测追问**：HR 接下来最可能问的 2-3 个问题及风险等级。
* **危险区域**：你应该主动避开的话题方向。
* **局势胜率**：一个 0-100% 的数字，反映当前对话走势对你的有利程度。

这些信息不会替换原有的实时建议，而是作为补充叠加显示。

### 怎么开启

1. 在 `.env` 文件里设置：

   ```
   MCTS_ENABLED=true
   ```

2. （可选）如果你有本地或远程的轻量模型可用，配上 Rollout 模型会让推演质量更高：

   ```
   MCTS_ROLLOUT_MODEL=qwen2.5-1.5b-instruct
   MCTS_ROLLOUT_API_BASE=http://localhost:8080/v1
   MCTS_ROLLOUT_API_KEY=your-key
   ```

   不配 Rollout 模型也能用，系统会自动降级到纯评分模式（速度更快，但推演质量略低）。

3. 其他可调参数（一般不需要改）：

   | 变量 | 默认值 | 说明 |
   |------|--------|------|
   | `MCTS_ITERATIONS` | 8 | 每轮搜索迭代次数 |
   | `MCTS_BRANCH_FACTOR` | 3 | 每步扩展的候选分支数 |
   | `MCTS_ROLLOUT_DEPTH` | 2 | 模拟对话深度（轮数） |
   | `MCTS_C_PUCT` | 1.4 | 探索系数，越大越倾向尝试新路径 |
   | `MCTS_SEARCH_TIMEOUT` | 25 | 单次搜索最大耗时（秒） |

4. 重启后端服务使配置生效。

### 不开启时有什么影响

没有。`MCTS_ENABLED` 默认为 `false`，不配置时系统行为与之前完全一致，不会引入额外延迟或资源消耗。

### 使用建议

* 如果你的机器没有 GPU 也没有轻量模型 API，先只设 `MCTS_ENABLED=true` 试试纯评分模式，它几乎不占额外资源。
* Rollout 模型推荐用参数量 1.5B-8B 的指令微调模型，太大会拖慢搜索。
* 胜率数字是统计估算，不是精确预测。它的价值在于趋势变化——连续下降说明你正在被引向弱点区域。
